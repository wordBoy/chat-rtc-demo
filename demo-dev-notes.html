Demo Development Notes

Demos are a lot of fun to create, and I always learn at least a baker's dozen new things. As a result, here's some notes that you may find useful.

Bootstrap Version 3

We use a modified version of Bootstrap - the getBootstrap.com version - for our demo.

We use the basic framework, but so far have avoided a lot of the components because we haven't vetted them for accessibility.

Notes on Making Bootstrap More Accessible

Note we did not say it's accessible...not ready to say that yet.

The coders have really taken out the focus rect - we put it back in everywhere we could find. In our bootstrap.css for this demo, see the /*chat-demo-a11y-fix*/ text to see where we've twiddled with something. For example, this seems to be why button focus rects do not appear in Firefox: 

button::-moz-focus-inner,
input::-moz-focus-inner {
  padding: 0;
  border: 0;
}

Android Chrome browser 7.x font scaling

Spent some time noodling on font scaling. I created a text size test page.

Get to this feature by:

  1. Open Chrome browser
  2. Go to the Options widget and choose Settings
  3. In settings, choose Accessibility
  4. The "Text scaling" is one of two available options for me.

I've encountered plenty of odd behavior with this feature. In particular:

  1. Small amounts of text on a page simply will not scale.
  2. The CSS property -webkit-text-size-adjust will not scale past 100%.
